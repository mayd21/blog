滚动导航设计

设计方案

在事件详情页面，需要对详情内容进行导航控制，如下图所示详情内容分为 Details、Participants 和 Comments 三个部分，需要实现点击对应的按钮跳转到相应的内容，同时滚动时根据内容的变化更新导航栏的状态。

  • 通过scrollTo方法将视图滚动到指定的位置
  • 通过监听onScroll事件控制导航栏的状态

实现过程

点击导航栏定位内容

需要计算各个需要定位的 DOM 在视图中的位置，从而进行视图滚动的控制，如下图所示，
通过getBoundingClientRect()方法可以计算出当前 DOM 相对于可视窗口的位置。

  • 计算出需要定位 DOM 的top距离
  • 通过scrollTo方法将滚动 DOM 向上滚动top高度即可

滚动内容控制导航栏

  • 监听onScroll事件，在滚动过程中计算 DOM 的位置
  • 若某 DOM 位置已经处于顶部在调整导航栏状态

存在的问题
若是直接按照上面逻辑进行计算当单击导航按钮时会进行滚动，从而触发滚动事件，滚动事件会计算 DOM 的位置同时调整导航栏的状态，这会导致点击导航栏按钮后，导航栏的状态会根据滚动的不断变化，这不是我们想要看到的。

我们需要知道当前的滚动是点击导航栏按钮scrollTo所导致的滚动还是用户触发的滚动。由于scrollTo并无法获取滚动结束的状态，因此需要在滚动过程中加入判断。
如下图所示为判断的逻辑：

  • 在点击导航按钮时标记点击的状态
  • 在滚动监听事件内判断是否滚动到顶部，并且是否有点击导航栏的按钮
  • 若滚动了底部或者滚到了导航栏所点击的 DOM 处，则判断点击导航动作结束
  • 取消导航按钮点击的标记

通过以上逻辑的判断可以在点击导航按钮的滚动过程中过滤 DOM 位置的计算，避免导航按钮的来回跳动。
